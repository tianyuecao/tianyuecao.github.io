Mask R-CNN代码阅读（一）

发表于 2018-05-21 |

国内自动驾驶创业公司图森未来对ICCV 2017最佳论文《Mask R-CNN》进行了完整复现，开源代码在Github上公布。（Mask R-CNN的mxnet实现）

代码结构

代码的主要结构如下：

  • mx-maskrcnn
    • data(储存训练和测试数据，这里是用了cityscapes的数据)
    • incubator-mxnet(mxnet源代码)
    • model(储存模型)
    • rcnn(主要的实现模块，后面主要对这部分进行分析)
      • core
      • CXX_OP
      • cython
      • dataset
      • io
      • processing
      • PY_OP
      • pycocotools
      • symbol
      • tools
      • utils
      • config.py(设定各种参数)
    • scripts(运行的脚本)
    • demo_mask.py(demo的main文件)
    • eval_maskrcnn.py(evaluate的main文件)
    • train_alternate_mask_fpn.py(训练的main文件)
    • Makefile(make cython和pycocotools)

rcnn部分

下面对rcnn部分代码进行分析。

config.py

config.py进行了各种参数设定，包括config和default两种参数的设定。在config参数设定中，分别设定了:

  • 网络相关参数: PIXEL_MEANS, ROIALIGN(True or False), RPN_FEAT_STRIDE, RCNN_FEAT_STRIDE, FIXED_PARAMS, FIXED_PARAMS_SHARED
  • 数据及相关参数: NUM_CLASSES, SCALES, ANCHOR_SCALES, ANCHOR_RATIOS, NUM_ANCHORS, CLASS_ID
  • 网络训练相关参数:
    • 整体网络: BATCH_IMAGES, ASPECT_GROUPING(True or False), SCALE(True or False), SCALE_RANGE
    • R-CNN: BATCH_ROIS, rois sampling params(FG_FRACTION, FG_THRESH, BG_THRESH_HI, BG_THRESH_LO), bounding vox regression params(BBOX_REGRESSION_THRESH, BBOX_WEIGHTS)
    • RPN:
      • RPN anchor loader: anchors batch size(RPN_BATCH_SIZE), anchors sampling params(RPN_FG_FRACTION, RPN_POSITIVE_OVERLAP, RPN_NEGATIVE_OVERLAP, RPN_CLOBBER_POSITIVES), bounding box regression params(RPN_BBOX_WEIGHTS, RPN_POSITIVE_WEIGHT)
      • RPN proposal: RPN_NMS_THRESH, RPN_PRE_NMS_TOP_N, RPN_POST_NMS_TOP_N, RPN_MIN_SIZE
    • approximate bounding box regression: BBOX_NORMALIZATION_PRECOMPUTED, BBOX_MEANS, BBOX_STDS
  • 测试相关参数

在default参数设定中，使用resnet_fpn网络，pretrained resnet-50模型。

另外，进行了network和dataset的参数设定。

最后用generate_config函数进行参数设定。

utils

进行模型和数据的储存和读入。

save_model.py

将模型数据checkpoint储存到文件。

load_model.py

加载模型。

combine_model.py

将两个模型checkpoint结合并储存到新的文件中。

load_data.py

加载数据，定义了以下几个函数：

  • load_gt_roidb：加载ground truth roidb
  • load_proposal_roidb：加载proposal roidb(训练时的append_gt)
  • merge_roidb：把roidb list concat到一起
  • filter_roidb：过滤出可以使用的roidb entries（至少有一个fg或bg roi的图片）

dataset

对数据集进行处理。

imdb.py

定义了一个image database类，产生一个图像相对路径的listimage_set_index和转化成的图像绝对路径。对于训练来说，需要将ground truth和proposals充分的混合。

cityscape.py

定义了一个继承了IMDB类的Cityscape子类，用来对cityscapes的数据进行处理。

在cityscape.py中具体实现了load_image_set_index(self)，image_path_from_index(self, index)和gt_roidb(self)， 定义并实现了load_from_seg(self, ins_seg_path)和load_citysacpe_annotations(self)等数据集专有的函数。另外，append_flipped_images等函数实现与IMDB有所区别。

CXX_OP

这部分用c语言实现了RoIAlign，这里不进行详细的分析。

pycocotools

这部分是与coco相关的部分，暂不分析。

cython

这部分是cython相关的部分，也不做分析。

这部分就先讨论到这里，剩下的模块（core, io, processing, PY_OP, symbol, tools）下次进行分析。
