Dynamic Programming 3 -- 背包问题

发表于 2017-11-03 |

背包问题是动态规划里非常典型的问题，从大一开始学python，到之后的C++再到后来的数据结构，基本上每年都会遇到一遍，这次算法又遇到了一次，就把背包问题再复习一遍。

问题描述

  • 有n个物体和一个背包；
  • 第i个物体的重量为wi > 0，价值为vi > 0；
  • 背包总承重W；
  • GOAL：使得装进背包的物品价值最大。

问题分析

首先考虑的是能不能用Greedy Algorithm求解，但是无论是以最大价值、最小重量还是价值和重量的比值为Greedy Patten，都不能满足最优。所以考虑使用动态规划。

首先定义OPT函数，由于这是一个binary choice的问题，所以首先考虑的是OPT(i) = 从1到i的最大价值subset。

  • OPT不选择i：OPT选择从1到i-1的最优值。
  • OPT选择i：选择了物体i不一定会是最优解，可能还能接受其他的物品，但是我们不知道我们是不是在选择了i之前还选择了其他的物品，甚至不知道有没有i的空间。

根据上面的分析，我们发现OPT如果只是关于i的函数，是不能解决这个问题的，所以我们增加一个参数限制，w。

OPT(i,w) = 从1到i且重量小于w的最大价值的subset.

  • OPT不选择i：OPT选择从1到i-1的重量小于等于w的OPT结果；
  • OPT选择I：OPT选择从1到i-1的重量小于等于w-wi的结果。

则OPT函数的定义为：

  1. 当i=0时，OPT(i,w) = 0；
  2. 当wi > w时，OPT(i,w) = OPT(i-1, w)；
  3. 否则，OPT(i,w) = max(OPT(i-1,w), vi + OPT(i-1,w-wi)).

通过OPT可以找到最大价值，为了找到选择的物品，从二维的memory中从右下角向上向左找解，向上的储存值与当前相等时说明没有选择此物品，直至不相等选择此物品，w变为w-wi，具体过程如下图：

算法流程

下面是一个bottom-up的解决方案：

KNAPSACK(n, W, w1, …, wn, v1, …, vn)

For w = 0 to W
M[0, w] = 0.

For I = 1 to n
For w = 1 to W
If (wi > w) M[i, w] = M[i-1, w].
Else M[i, w] = max(M[i-1, w], vi + M[i-1, w-wi]).

Return M[n, W].

复杂度分析

填表时每填一格需要时间为O(1)，表格共有O(nW)格，所以时间复杂度为O(nW)，空间复杂度也为O(nW).
